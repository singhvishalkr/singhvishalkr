# Vishal Kumar Singh

I build backends where contracts stay stable while everything underneath moves: JDK generations, HTTP stacks, dependency graphs, and where secrets actually resolve.

Day to day that is a large B2B corporate-travel platform at **MakeMyTrip** - Java **21**, **Spring Boot 4** on **Spring Framework 7**, **Jackson 3**, heavy REST traffic, search and booking paths, and enterprise policy in the middle. The interesting problems are almost never greenfield syntax; they are freezing behaviour for consumers while you swap frameworks, trim transitive CVE surfaces, and wire traces so on-call can trust what they see.

### What I spend cycles on

- **Migrations that do not become rewrites.** Same verbs and payloads on the wire; different runtime underneath. Problem payloads shaped so clients can branch on structure, not string hacks.
- **Thin seams at boundaries.** Cloud secret backends and HTTP clients stay behind interfaces so environments can diverge without cloning services.
- **Operability baked in.** Health, metrics, and traces aligned so deploys and incidents read from one story.
- **Supply chain discipline.** Fewer mystery JARs, fewer scanner fires, smaller binaries where it matters.
